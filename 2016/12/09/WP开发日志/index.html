2016-12-09

VPS, WordPress, 教程

一、服务器篇

1. 购置服务器

引用百度百科的说明

服务器，也称伺服器，是提供计算服务的设备。由于服务器需要响应服务请求，并进行处理，因此一般来说服务器应具备承担服务并且保障服务的能力。

不过就是一台电脑，所以如果条件允许，用自己闲置的电脑也是可以的，不过还是建议去租一个，如果非要把三年前换下来的电脑派上用场，遇到各种坑这里就没办法了。

普通用户可以直接找国内的服务商，比如阿里云。不过由于某些不可描述的原因，我还是找了国外的廉价服务器（其实是为了做不可描述的事，先租了服务器之后才想着弄个博客玩玩）。

2. 连接服务器

刚租下的服务器默认已经装好了系统，但是不一定是我们要的，所以需要在后台重装系统。
在重装系统的时候，有几个重要数据一定要记住：IP地址、SSH端口号、root密码。这些是将来操作服务器必备信息。

等服务器系统装好之后，使用SSH客户端（常用的有SecureCRT，但我在用另一款号称Windows神器的 cmder ，他基本上相当于Windows下的一个简易Linux系统，除了SSH还有其他常用工具，甚至支持Linux命令）连接服务器。

二、购买域名

只是出于“玩”的目的，建议找个免费的域名就行了，比较知名免费顶级域名是 .tk 域名，可以到 官网 申请，然后设置 DNS ，定期续约就可以了。

当然如果你愿意，每年花一笔钱买个常见的 .com 或者其他域名也可以。

三、配置环境 LNMP

1. 系统 Linux

这个好说，根据你找的服务商和套餐，服务器可选系统很多，从Linux到Windows应有尽有，不过听说有些服务器是限制重装次数的，对新玩家不太友好。我的服务器是使用 Centos 7 x86_64 minimal 系统，纯命令行，我后来从官网下载的 iso 文件只有400多MB。

初始化之后需要先更新系统
yum update -y

2. 一键完成NMP（nginx+MySQL+PHP）

使用LNMP.org提供的一键安装包，最新稳定版本是1.3。
wget -c http://soft.vpser.net/lnmp/lnmp1.3-full.tar.gz &amp;&amp; tar zxf lnmp1.3-full.tar.gz &amp;&amp; cd lnmp1.3-full &amp;&amp; ./install.sh lnmp
其中 && 表示连续执行命令，分别是下载完整包、解压缩、打开文件夹、执行安装脚本

中途会有一些其他提示


3. 添加虚拟主机

创建成功后，所有参数都保存在/usr/local/nginx/conf/vhost/域名.conf文件里，可以二次修改

到此，LMNP环境搭建成功，在浏览器输入服务器IP地址，可以看到搭建成功的信息

1. 安装 WordPress

  1. 到wordpress官网找到最新下载包
  2. 用 wget 命令下载
  3. 用 tar 命令解压
  4. 用 mv 命令移动到/home/wwwroot/default目录下
  5. 把目录下的index.html删除
  6. 访问服务器，会打开WordPress初始化页面
    !!!我第一次下载的是英文版，这里就用英文版的截图了，中文官网下载的是中文版的!!!
  7. 填入必要信息，完成初始化

    3) 走上 https 之路

    使用免费的Let’s encrypt，参考网站

    1. wget https://dl.eff.org/certbot-auto
      chmod a+x ./certbot-auto
      ./certbot-auto –help
    2. service nginx stop
    3. ./certbot-auto certonly –standalone –agree-tos -v -t –email tianwei1987@gmail.com -d woodygo.tk
      成功后证书文件放在了/etc/letsencrypt/live下。

    4. 修改 /usr/local/nginx/conf/vhost/woodygo.tk.conf ，添加https 信息

!!!一定要有最后两行，不然跳转之后不能运行 php 文件，旧的文章 404 !!!
并在旧的 server 下添加return 301 https://$server_name$request_uri;

会遇到的坑

Q1：

  • Unable to create directory wp-content/uploads/2016/12. Is its parent directory writable by the server?
  • 安装/更新/卸载 插件时提示需要输入FTP信息
    Answer
    chown www:www -R /home/wwwroot/woodygo.tk/
    chown -R 755 /home//wwwroot/woodygo.tk/

    Q2：

    WordPress后台主题中只显示一个主题不显示其他主题
    Answer
    1. vi /usr/local/php/etc/php.ini
    2. 删除 disable_funcions 后边的 scandir
    3. 重启 php-fpm 服务
