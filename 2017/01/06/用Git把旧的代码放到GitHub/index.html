用Git把旧的代码放到GitHub

2017-01-06
科技
GIT, GitHub, 教程

前提

  • 在本地有写了一半的代码
  • 已经有了GitHub帐号
  • 使用cmder，这样就不用下载安装GIT

    目的

    这次就是初步使用github，然后利用一个demo代码体验GIT和GitHub，为以后把博客放到GitHub上做准备，还有做到跨平台的文档管理（这点可能还会进化成使用自己的服务器）。

    步骤

  1. 用cmder进入代码文件夹
  2. git init  //初始化
  3. ssh-keygen -t rsa -C tianwei1987@gmail.com   //生成SSH密钥
  4. 生成密钥过程中会提示设置路径和密码，这里全部直接跳过了
  5. 根据成功提示，找到 c:/Users/Woody/.ssh/id_rsa.pub 文件
  6. 到GitHub个人设置里找到SSH and GPG keys，并把刚才文件里的SSH密钥添加到后台
  7. ssh -T git@github.com  //验证是否成功了，需要看看提示，输个yes
  8. You’ve successfully authenticated, but GitHub does not provide shell access  //这个是显示的，表示授权成功，但是还需要设置帐号
  9. git config –global user.name johngoofy //设置用户名
  10. git config –global user.email tianwei1987@gmail.com //设置邮箱
  11. 登录GitHub，打开一个空的responsitory，找到Clone地址然后切换成“Clone with SSH”，然后复制下来
  12. git remote add origin git@github.com:johngoofy/gs.app.git  //连接
  13. git add .  //把当前文件夹全加入列表
  14. git commit -m “first”  //把当前的提交标记为“first”
  15. git push origin master  //上传

本期完结，待续
